## üõ†Ô∏è Generation and Usage

### Automatic SVG Generation

Use the provided PowerShell script to convert all Mermaid files to SVG format:

```powershell
# Convert all .mmd files in current directory
.\generate-svg.ps1

# Show detailed output
.\generate-svg.ps1 -Verbose

# Preview what would be converted (dry run)
.\generate-svg.ps1 -WhatIf

# Specify custom directories
.\generate-svg.ps1 -InputPath ".\source" -OutputPath ".\output"
```

### Manual Conversion

If you need to convert individual files:

```bash
# Install Mermaid CLI (one-time setup)
npm install -g @mermaid-js/mermaid-cli

# Convert individual files
mmdc -i project-dependency-diagram.mmd -o project-dependency-diagram.svg
mmdc -i class-diagram-domain-entities.mmd -o class-diagram-domain-entities.svg
# ... etc for each file
```

### Prerequisites

- **Node.js** and **npm** installed
- **Mermaid CLI**: `npm install -g @mermaid-js/mermaid-cli`
- **PowerShell** (for the automation script)

When the application evolves, update the corresponding diagrams:

1. **Edit** the `.mmd` source files
2. **Run** `.\generate-svg.ps1` to regenerate SVGs
3. **Commit** both `.mmd` and `.svg` files to source control
4. **Review** diagrams match current implementation

## üìê Diagram Standards

### Mermaid Syntax Guidelines
- Use consistent naming conventions
- Include descriptive comments at the top
- Apply semantic styling with CSS classes
- Group related elements in subgraphs

### Visual Consistency
- **Colors**: Use semantic color coding (success=green, error=red, etc.)
- **Shapes**: Standard shapes for different element types
- **Layout**: Logical flow from top-to-bottom or left-to-right
- **Labels**: Clear, concise, and descriptive
